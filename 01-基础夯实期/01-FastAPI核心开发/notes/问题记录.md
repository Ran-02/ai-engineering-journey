## 详细问题与修改

### 1. 分页依赖注入未启用

- **问题**：`get_pagination` 函数被整段注释，未投入使用。任务书第 4 节明确要求"定义分页依赖函数 get_pagination，统一处理分页参数校验"。
- **修改**：取消注释，激活 `get_pagination`；`GET /tasks` 改用 `Depends(get_pagination)` 注入分页参数。

```python
# 修改前（注释状态）
# def get_pagination(skip:int=Query(default=0,ge=0), ...):
#     return {"skip":skip,"limit":limit}
# def read_tasks(Query: TaskQuery):
#     ...

# 修改后
def get_pagination(
    skip: int = Query(default=0, ge=0, description="跳过的记录数"),
    limit: int = Query(default=10, ge=1, le=100, description="每页记录数"),
):
    return {"skip": skip, "limit": limit}

def read_tasks(pagination: dict = Depends(get_pagination)):
    skip = pagination["skip"]
    limit = pagination["limit"]
    return tasks_db[skip : skip + limit]
```

### 2. 参数命名冲突

- **问题**：`read_tasks(Query: TaskQuery)` 和 `read_task(Path: TaskPath)` 中，参数名 `Query`、`Path` 与 `from fastapi import Query, Path` 重名。参数遮蔽了导入，后续代码若想使用 fastapi 的 `Query()`/`Path()` 将无法访问。
- **修改**：消除参数命名冲突，统一使用语义化参数名。

### 3. PUT/DELETE 缺失路径参数校验

- **问题**：GET /tasks/{task_id} 通过 `TaskPath` 模型做了 `gt=0` 校验，但 PUT 和 DELETE 直接用 `task_id: int`，未做任何校验。任务书要求"路径参数 task_id 限制必须大于 0"。
- **修改**：PUT、DELETE 统一加上 `Path(..., gt=0, description="任务ID必须大于0")`。

```python
# 修改前
def update_task(task_id: int, task: TaskCreate):
    ...

# 修改后
def update_task(
    *,
    task_id: int = Path(..., gt=0, description="任务ID必须大于0"),
    task: TaskCreate = Body(...),
):
    ...
```

### 4. PUT 接口 body 参数默认值不当

- **问题**：`task: TaskCreate = None` 作为默认值，若请求未传 body，FastAPI 会注入 `None`，后续 `task.model_dump()` 将抛出 `AttributeError`。
- **修改**：使用 `Body(...)` 作为默认值，既满足 Python 语法（参数需有默认值），又明确告知 FastAPI 该 body 为必填。

### 5. 死代码清理

- **问题**：文件中残留被注释的 `get_pagination` 实现、未使用的 `TaskQuery` 模型、`TaskPath` 模型（仅 GET 使用，PUT/DELETE 未复用）。
- **修改**：删除 `TaskQuery`、`TaskPath` 类定义；改用标准 FastAPI 的 `Path()`/`Query()` 直接标注在函数参数上；移除注释代码。

### 6. 模型构建方式优化

- **问题**：`create_task` 和 `update_task` 中手动逐个展开字段：
  ```python
  Task(task_id=next_id, title=task.title, description=task.description, priority=task.priority)
  ```
- **修改**：使用 `**task.model_dump()` 自动展开，减少字段变更时的遗漏风险：
  ```python
  Task(task_id=next_id, **task.model_dump())
  ```
