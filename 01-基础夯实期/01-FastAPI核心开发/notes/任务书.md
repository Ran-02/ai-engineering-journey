# 第1周实战任务书

## 一、任务说明
- 项目目标：基于内存存储实现完整 RESTful 任务管理接口，落地 FastAPI 基础核心知识点
- 存放路径：`01-基础夯实期/01-FastAPI核心开发/code/`
- 核心要求：独立编码完成，全场景测试，无未捕获异常

## 二、核心任务清单
### 1. 环境初始化
1. 创建 `main.py` 入口文件
2. 安装依赖：`fastapi`、`uvicorn`
3. 初始化 FastAPI 应用实例，配置标题与描述
4. 实现根路径欢迎接口
5. 启动服务，验证 `/docs` 交互式文档可正常访问

### 2. 完整 CRUD 接口实现
1. 定义 Pydantic 模型：`TaskCreate`（请求体）、`Task`（响应体，含 task_id）
2. 使用内存列表存储数据，实现自增 ID 计数器
3. 实现 5 个标准接口：
   - `POST /tasks`：创建任务，成功返回 201 状态码
   - `GET /tasks`：分页查询任务列表，支持 skip、limit 参数
   - `GET /tasks/{task_id}`：查询单个任务详情
   - `PUT /tasks/{task_id}`：更新指定任务
   - `DELETE /tasks/{task_id}`：删除指定任务，成功返回 204 状态码
4. 资源不存在时抛出 404 业务异常

### 3. 三级参数校验
1. 路径参数 `task_id`：限制必须大于 0
2. 查询参数：`skip` ≥ 0（默认0），`limit` 范围 1-100（默认10）
3. 请求体字段校验：
   - `title`：必填，长度 1-50
   - `priority`：整数，范围 1-5
   - `description`：可选，最大长度 500

### 4. 依赖注入抽离
1. 定义分页依赖函数 `get_pagination`，统一处理分页参数校验
2. 任务列表接口通过 `Depends` 注入使用
3. 功能与原分页逻辑完全一致

### 5. 全量测试与提交
1. 覆盖所有正常场景与异常场景测试
2. 确认无未捕获 500 错误
3. 按规范提交代码至 GitHub 对应目录

## 三、验收标准
✅ 服务可正常启动，交互式文档可正常访问
✅ 5 个 CRUD 接口功能全部正常
✅ 路径、查询、请求体三级校验全部生效，非法参数返回 422
✅ 资源不存在统一返回 404 异常
✅ 创建返回 201、删除返回 204，状态码语义正确
✅ 分页逻辑通过依赖注入实现，代码无重复
✅ 所有边界场景无崩溃、无未捕获异常

## 四、运行方式
1. 安装依赖：`pip install fastapi uvicorn`
2. 启动服务：`uvicorn main:app --reload`
3. 接口文档：http://127.0.0.1:8000/docs
