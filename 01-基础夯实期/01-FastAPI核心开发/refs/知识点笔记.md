## 一、FastAPI 基础与环境搭建

### 1.1 安装

```bash
pip install fastapi uvicorn
```

| 包 | 用途 |
|---|---|
| `fastapi` | Web 框架，提供路由、校验、OpenAPI 文档等核心能力 |
| `uvicorn` | ASGI 服务器，负责运行 FastAPI 应用 |

### 1.2 最简应用骨架

```python
from fastapi import FastAPI

app = FastAPI(title="我的API", description="描述", version="1.0.0")

@app.get("/")
def root():
    return {"message": "Hello World"}
```

### 1.3 启动方式

```bash
# 终端直接启动
uvicorn main:app --reload

# 代码内启动（适用于调试）
if __name__ == "__main__":
    import uvicorn
    uvicorn.run("main:app", reload=True)
```

| 参数 | 说明 |
|---|---|
| `main:app` | `main.py` 文件中的 `app` 实例 |
| `--reload` | 热重载，代码修改后自动重启（开发用） |

### 1.4 交互式文档

启动后自动生成：
- **Swagger UI** → `http://127.0.0.1:8000/docs`
- **ReDoc** → `http://127.0.0.1:8000/redoc`

> **关键理解**：交互式文档是 FastAPI 的"杀手特性" — 基于 OpenAPI 规范自动生成，无需手动维护接口文档。

### 1.5 路由装饰器与 HTTP 方法

```python
@app.get("/")         # GET 请求
@app.post("/")        # POST 请求，创建资源
@app.put("/")         # PUT 请求，更新资源
@app.delete("/")      # DELETE 请求，删除资源
```

常用装饰器参数：

| 参数 | 用途 | 示例 |
|---|---|---|
| `summary` | 接口简短标题 | `summary="创建任务"` |
| `description` | 接口详细描述 | `description="创建一个新的任务项"` |
| `status_code` | 成功响应状态码 | `status_code=201` |
| `tags` | 接口分组标签 | `tags=["任务管理"]` |

## 二、路径参数 (Path Parameters)

### 2.1 基本用法

路径参数是 URL 路径的一部分，用于定位**单个资源**。

```python
@app.get("/items/{item_id}")
def read_item(item_id: int):      # 自动转换为 int 类型
    return {"item_id": item_id}
```

### 2.2 类型自动转换

FastAPI 会根据类型注解自动解析路径参数：

| 声明类型 | 转换行为 | 无效输入 |
|---|---|---|
| `int` | 自动转整数 | 返回 422 |
| `float` | 自动转浮点 | 返回 422 |
| `str` | 保持字符串 | — |
| `bool` | 自动转布尔 | 返回 422 |

### 2.3 枚举限制（预设值）

```python
from enum import Enum

class Status(str, Enum):
    active = "active"
    completed = "completed"

@app.get("/tasks/{status}")
def filter_by_status(status: Status):
    return {"status": status}
```

### 2.4 路径顺序

FastAPI 按路由定义顺序匹配，**固定路径应放在参数化路径之前**：

```python
@app.get("/users/me")     # 优先匹配
@app.get("/users/{user_id}")  # 后匹配
```

## 三、查询参数 (Query Parameters)

### 3.1 基本用法

查询参数是 URL 中 `?` 后面的键值对，用于**过滤、排序、分页**。

```python
@app.get("/items")
def read_items(skip: int = 0, limit: int = 10):
    return items_db[skip : skip + limit]
```

### 3.2 核心规则

| 规则 | 说明 | 示例 |
|---|---|---|
| **可选参数** | 有默认值 → 查询参数可选 | `skip: int = 0` |
| **必填参数** | 无默认值 → 查询参数必填 | `q: str` |
| **类型转换** | 自动转换为声明类型 | `"10"` → `10` |
| **可选 + None** | 未传时值为 `None` | `q: str \| None = None` |

### 3.3 路径参数 vs 查询参数

```
GET /tasks/5           → task_id = 5（路径参数，定位单个资源）
GET /tasks?skip=0&limit=10  → skip=0, limit=10（查询参数，过滤集合）
```

| | 路径参数 | 查询参数 |
|---|---|---|
| 位置 | URL 路径中 | `?` 之后 |
| 用途 | 定位单个资源 | 过滤、分页、排序 |
| 必填 | 通常是 | 通常可选 |
| 示例 | `/users/{id}` | `?page=1&size=20` |

## 四、数值校验：Path & Query

### 4.1 `Path` 校验器

```python
from fastapi import Path

@app.get("/items/{item_id}")
def read_item(
    item_id: int = Path(..., gt=0, description="物品ID必须大于0"),
):
    ...
```

### 4.2 `Query` 校验器

```python
from fastapi import Query

@app.get("/items")
def read_items(
    skip: int = Query(default=0, ge=0, description="跳过的记录数"),
    limit: int = Query(default=10, ge=1, le=100, description="每页记录数"),
):
    ...
```

### 4.3 校验参数速查

| 参数 | 含义 | 使用场景 |
|---|---|---|
| `gt` | 大于 (greater than) | `gt=0` — ID 必须为正数 |
| `ge` | 大于等于 (greater or equal) | `ge=0` — skip 可为零 |
| `lt` | 小于 (less than) | `lt=100` |
| `le` | 小于等于 (less or equal) | `le=100` — limit 上限 |
| `min_length` | 最小字符串长度 | `min_length=1` |
| `max_length` | 最大字符串长度 | `max_length=50` |
| `default` | 默认值 | 参数未传时使用 |
| `description` | 参数描述，显示在文档中 | `description="任务标题"` |
| `...` (Ellipsis) | 标记为必填 | `Path(...)` — 路径参数通常是必填的 |

### 4.4 必填 vs 可选

```python
# 必填（路径参数通常必填）
item_id: int = Path(..., gt=0)

# 可选 + 默认值
skip: int = Query(default=0, ge=0)

# 可选 + None（不传则不限定）
q: str | None = Query(default=None, max_length=100)
```

> **`...` (Ellipsis)** 表示"此参数没有默认值，是必需的"。注意它与 `Query()` / `Path()` 配合使用时的语义：`Path(...)` = 路径参数且必填，`Query(...)` = 查询参数且必填。

## 五、请求体与 Pydantic 模型

### 5.1 定义数据模型

```python
from pydantic import BaseModel, Field

class TaskCreate(BaseModel):
    title: str = Field(..., min_length=1, max_length=50, description="任务标题")
    description: str | None = Field(default=None, max_length=500, description="任务描述")
    priority: int = Field(default=1, ge=1, le=5, description="优先级 1-5")
```

### 5.2 请求体模型 vs 响应体模型

推荐做法：**请求和响应使用不同的模型**，避免将 ID 等敏感字段暴露给输入。

```python
class TaskCreate(BaseModel):     # 请求体：不含 ID
    title: str
    priority: int

class Task(TaskCreate):           # 响应体：含 ID（继承请求体）
    task_id: int
```

### 5.3 Field 校验规则速查

| 参数 | 用途 | 示例 |
|---|---|---|
| `...` | 必填字段 | `Field(...)` |
| `default` | 默认值 | `Field(default=0)` |
| `default=None` | 可选字段 | `Field(default=None)` |
| `min_length` | 最小字符数 | `Field(min_length=1)` |
| `max_length` | 最大字符数 | `Field(max_length=50)` |
| `gt` | 大于 | `Field(gt=0)` |
| `ge` | 大于等于 | `Field(ge=1)` |
| `le` | 小于等于 | `Field(le=5)` |
| `description` | 字段描述 | `Field(description="任务标题")` |

### 5.4 模型在接口中的使用

```python
@app.post("/tasks", status_code=201)
def create_task(task: TaskCreate):           # 自动从请求体解析
    new_task = Task(task_id=next_id, **task.model_dump())
    return new_task
```

### 5.5 `model_dump()` 方法（Pydantic v2）

将 Pydantic 模型实例转换为字典：

```python
task = TaskCreate(title="A", priority=3)
task.model_dump()  # {"title": "A", "description": None, "priority": 3}
```

> **注意**：Pydantic v2 使用 `model_dump()`，v1 使用 `.dict()`。

## 六、错误处理与 HTTP 异常

### 6.1 基础异常抛出

```python
from fastapi import HTTPException

@app.get("/items/{item_id}")
def read_item(item_id: int):
    for item in items_db:
        if item.id == item_id:
            return item
    raise HTTPException(status_code=404, detail="物品未找到")
```

### 6.2 HTTPException 参数

| 参数 | 说明 | 示例 |
|---|---|---|
| `status_code` | HTTP 状态码 | `404` |
| `detail` | 错误描述信息 | `"任务未找到"` |
| `headers` | 自定义响应头（可选） | `{"X-Error": "..."}` |

### 6.3 常见异常场景

```python
# 404 — 资源不存在
raise HTTPException(status_code=404, detail="任务未找到")

# 400 — 请求参数错误
raise HTTPException(status_code=400, detail="无效的请求参数")
```

### 6.4 FastAPI 自动返回的错误

| 场景 | 状态码 | 说明 |
|---|---|---|
| 请求体验证失败 | **422** | 字段类型错误、长度超限、数值越界等 |
| 路径参数格式错误 | **422** | 如 `int` 参数收到非数字字符 |
| 查询参数越界 | **422** | 如 `limit` 超出 `le=100` 范围 |
| 资源不存在（手动） | **404** | 由业务代码中抛出 |
| 未捕获异常 | **500** | 服务端内部错误（应避免） |

## 七、依赖注入 (Dependency Injection)

### 7.1 为什么需要依赖注入

将**公共逻辑**（分页、认证、数据库会话等）抽离为可复用的函数，避免在每个接口中重复实现。

### 7.2 定义依赖函数

```python
from fastapi import Depends, Query

def get_pagination(
    skip: int = Query(default=0, ge=0, description="跳过的记录数"),
    limit: int = Query(default=10, ge=1, le=100, description="每页记录数"),
):
    return {"skip": skip, "limit": limit}
```

### 7.3 在接口中注入

```python
@app.get("/tasks")
def read_tasks(pagination: dict = Depends(get_pagination)):
    skip = pagination["skip"]
    limit = pagination["limit"]
    return tasks_db[skip : skip + limit]
```

### 7.4 依赖注入的执行逻辑

```
客户端请求 → FastAPI 识别 Depends → 调用 get_pagination(skip, limit)
           → 将返回值注入 read_tasks 的 pagination 参数
           → 执行接口逻辑 → 返回响应
```

### 7.5 依赖注入的优势

| 优势 | 说明 |
|---|---|
| **复用性** | 多个接口共享同一套校验逻辑 |
| **可测试性** | 依赖可 mock，单元测试更简单 |
| **声明式** | 参数校验与业务逻辑分离 |
| **文档** | 依赖中的参数会自动显示在 OpenAPI 文档中 |

### 7.6 依赖注入的多种形式

```python
# 1. 函数作为依赖（最常用）
def common_params(q: str | None = None, skip: int = 0, limit: int = 100):
    return {"q": q, "skip": skip, "limit": limit}
@app.get("/items")
def read_items(params: dict = Depends(common_params)):
    ...

# 2. 类作为依赖
class CommonQuery:
    def __init__(self, q: str | None = None, skip: int = 0, limit: int = 100):
        self.q = q
        self.skip = skip
        self.limit = limit
@app.get("/items")
def read_items(query: CommonQuery = Depends()):
    ...
```

## 八、RESTful 设计规范

### 8.1 资源导向设计

RESTful API 的核心思想是**面向资源**，而非面向操作。

```
❌ /createTask       → 动词，不是 RESTful
❌ /getTask?id=5     → 动词 + 查询参数
❌ /deleteTask/5     → 动词
✅ POST   /tasks           → 创建任务
✅ GET    /tasks           → 查询任务列表
✅ GET    /tasks/{id}      → 查询单个任务
✅ PUT    /tasks/{id}      → 更新任务
✅ DELETE /tasks/{id}      → 删除任务
```

### 8.2 CRUD 与 HTTP 方法映射

| 操作 | HTTP 方法 | URL | 状态码 |
|---|---|---|---|
| 创建 (Create) | `POST` | `/tasks` | `201 Created` |
| 查询列表 (Read) | `GET` | `/tasks` | `200 OK` |
| 查询单个 (Read) | `GET` | `/tasks/{id}` | `200 OK` |
| 更新 (Update) | `PUT` | `/tasks/{id}` | `200 OK` |
| 删除 (Delete) | `DELETE` | `/tasks/{id}` | `204 No Content` |

### 8.3 命名规范

- 使用**复数名词**：`/tasks`、`/users`、`/items`
- 使用**小写字母 + 连字符**：`/task-items`
- 不使用动词：`/getTasks` ❌ → `/tasks` ✅
- 层级关系：`/users/{id}/tasks`

### 8.4 状态码语义速查

| 状态码 | 含义 | 使用场景 |
|---|---|---|
| **200** | OK — 请求成功 | GET、PUT 成功返回 |
| **201** | Created — 创建成功 | POST 创建资源后返回 |
| **204** | No Content — 无返回体 | DELETE 成功后返回 |
| **400** | Bad Request — 请求格式错误 | 参数不符合预期 |
| **404** | Not Found — 资源不存在 | 查询/更新/删除不存在的资源 |
| **422** | Unprocessable Entity — 校验失败 | Field/Path/Query 校验不通过 |
| **500** | Internal Server Error — 服务端异常 | 未捕获异常（应避免） |

## 九、Pydantic 字段校验进阶

### 9.1 常用 Field 约束

```python
from pydantic import BaseModel, Field

class Item(BaseModel):
    # 字符串约束
    name: str = Field(..., min_length=1, max_length=100)
    regex_field: str = Field(..., pattern=r"^[a-z]+$")
    
    # 数值约束
    price: float = Field(..., gt=0, le=10000)
    quantity: int = Field(default=0, ge=0)
    
    # 可选字段
    description: str | None = Field(default=None, max_length=500)
```

### 9.2 模型配置（Pydantic v2）

```python
class Task(BaseModel):
    task_id: int
    title: str
    
    model_config = {
        "from_attributes": True,   # 从 ORM 对象创建（SQLAlchemy 等）
        "json_schema_extra": {     # 自定义 JSON Schema
            "example": {
                "task_id": 1,
                "title": "示例任务"
            }
        }
    }
```

### 9.3 Pydantic v1 vs v2 主要变化

| 特性 | v1 | v2 |
|---|---|---|
| 转字典 | `.dict()` | `.model_dump()` |
| 转 JSON | `.json()` | `.model_dump_json()` |
| ORM 模式 | `class Config: orm_mode = True` | `model_config = {"from_attributes": True}` |
| 校验引擎 | 自研 | 基于 Rust 的 `pydantic-core` |
| 速度 | 较慢 | 快 5-50 倍 |

## 十、常见状态码速查

| 状态码 | 中文名 | 英文名 | 含义 |
|---|---|---|---|
| 200 | 成功 | OK | 请求正常处理 |
| 201 | 已创建 | Created | 资源创建成功 |
| 204 | 无内容 | No Content | 成功但无返回体 |
| 400 | 错误请求 | Bad Request | 请求语法错误 |
| 401 | 未授权 | Unauthorized | 需要身份认证 |
| 403 | 禁止 | Forbidden | 无权访问 |
| 404 | 未找到 | Not Found | 资源不存在 |
| 422 | 无法处理 | Unprocessable Entity | 请求体验证失败 |
| 500 | 内部错误 | Internal Server Error | 服务端异常 |
| 502 | 网关错误 | Bad Gateway | 上游服务异常 |
