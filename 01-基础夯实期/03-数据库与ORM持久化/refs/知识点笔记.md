## 一、ORM 核心思想与入门

### 1.1 什么是 ORM

**ORM** = Object Relational Mapping（对象关系映射）。

用一句话说就是：**用 Python 对象来操作数据库，不用写 SQL 语句**。

```python
# ❌ 原生 SQL：写字符串，容易出错
cursor.execute("INSERT INTO tasks (title, priority) VALUES ('买菜', 3)")

# ✅ ORM：操作 Python 对象，类型安全
task = Task(title="买菜", priority=3)
session.add(task)
session.commit()
```

### 1.2 ORM 的核心价值

| 对比 | 原生 SQL | ORM |
|---|---|---|
| 代码可读性 | SQL 字符串拼在 Python 里，混杂难读 | 纯 Python 对象操作，清晰 |
| 类型安全 | 值错了运行时才报错 | 赋值时就有类型提示 |
| 数据库迁移 | 手写 SQL 脚本 | Alembic 自动生成 |
| SQL 注入 | 容易写出漏洞 | 参数自动转义，天然免疫 |
| 切换数据库 | 需要改 SQL 方言 | 换连接地址即可 |

### 1.3 ORM 的核心映射逻辑

```
数据库表（tasks）          ORM 模型类（Task）
┌────────┬────────┐      ┌──────────────────┐
│ id     │ INT    │ ───→ │ id: int          │
│ title  │ VARCHAR│ ───→ │ title: str       │
│ priority│ INT   │ ───→ │ priority: int    │
└────────┴────────┘      └──────────────────┘
    表记录（一行）          Python 对象（一个实例）
    INSERT / SELECT        session.add() / session.query()
```

| 数据库概念 | ORM 对应 | 类比 |
|---|---|---|
| 表 (Table) | Model 类 | 一个 Excel 表格模板 |
| 列 (Column) | 类属性 | 表格的列名 |
| 记录 (Row) | 类的实例 | 表格中的一行数据 |
| 主键 (Primary Key) | `mapped_column(primary_key=True)` | 每行的唯一编号 |
| 事务 (Transaction) | Session | 一个"操作批次" |

### 1.4 最简 ORM 示例

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, Session

# 1. 创建引擎（连上哪个数据库）
engine = create_engine("sqlite:///./tasks.db", echo=True)

# 2. 定义 Base（所有模型的父类）
class Base(DeclarativeBase):
    pass

# 3. 定义模型（ORM 类 → 数据库表）
class Task(Base):
    __tablename__ = "tasks"          # 表名

    id: Mapped[int] = mapped_column(primary_key=True)  # 主键
    title: Mapped[str]               # 字符串，非空
    priority: Mapped[int]            # 整数

# 4. 建表（创建表结构）
Base.metadata.create_all(engine)

# 5. 创建会话（开始"对话"）
with Session(engine) as session:
    # 6. 增：添加一条记录
    task = Task(title="买菜", priority=3)
    session.add(task)
    session.commit()                 # 提交事务，写入数据库

    # 7. 查：查询所有记录
    all_tasks = session.query(Task).all()
    for t in all_tasks:
        print(t.title)
```

## 二、SQLAlchemy 三大核心组件

### 2.1 总览

SQLAlchemy 2.0 的三个核心组件，记住这个比喻：

| 组件 | 类比 | 职责 |
|---|---|---|
| **Engine** (引擎) | 快递总站 | 管理数据库连接和连接池 |
| **Session** (会话) | 快递员 | 每趟"对话"负责增删改查和事务 |
| **Base** (基类) | 快递单模板 | 所有模型类的父类，定义表结构 |

### 2.2 Engine（引擎）

Engine 是数据库的"大门"，负责：
- 建立与数据库的连接
- 管理连接池（省去反复创建连接的开销）
- 将 ORM 操作翻译成 SQL 语句

```python
from sqlalchemy import create_engine

# SQLite（文件型数据库，开发用）
engine = create_engine("sqlite:///./tasks.db", echo=True)

# MySQL（需要安装 pymysql）
engine = create_engine("mysql+pymysql://user:password@host:3306/dbname")

# PostgreSQL（需要安装 psycopg2）
engine = create_engine("postgresql://user:password@host:5432/dbname")
```

| 参数 | 说明 | 示例 |
|---|---|---|
| `echo=True` | 打印执行的 SQL 语句（开发调试用） | 控制台会显示 `CREATE TABLE`、`INSERT INTO` 等 |
| `pool_size=5` | 连接池大小（默认 5） | 最多同时保持 5 个连接 |
| `max_overflow=10` | 连接池溢出上限 | 高峰期最多额外开 10 个连接 |
| `pool_pre_ping=True` | 使用前检查连接是否有效 | 避免使用已断开的连接 |

```python
# 带连接池配置的引擎
engine = create_engine(
    "sqlite:///./tasks.db",
    echo=True,
    pool_size=5,              # 连接池保持 5 个连接
    max_overflow=10,          # 峰值额外创建最多 10 个
    pool_pre_ping=True,       # 用前先 ping 一下
)
```

### 2.3 Base（基类）

Base 是所有数据库模型的"祖宗"。每个数据库表对应一个继承 Base 的类。

```python
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column

# 1. 创建一个 Base 基类（整个项目只需要这一个）
class Base(DeclarativeBase):
    pass

# 2. 定义模型（数据库表）- 每个表一个类，都继承 Base
class Task(Base):
    __tablename__ = "tasks"  # 数据库中的表名（必填）

    # 字段定义：属性名 = 列配置
    id: Mapped[int] = mapped_column(primary_key=True)  # 主键
    title: Mapped[str]                                  # 字符串（默认非空）
    description: Mapped[str | None]                     # 可选字符串
    priority: Mapped[int] = mapped_column(default=1)    # 整数，默认值 1

class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str]
```

**字段类型映射：**

| Python 类型 | 数据库类型 | 说明 |
|---|---|---|
| `Mapped[int]` | INTEGER | 整数 |
| `Mapped[str]` | VARCHAR | 字符串（默认非空） |
| `Mapped[str \| None]` | VARCHAR | 可选字符串（可为空） |
| `Mapped[float]` | FLOAT | 浮点数 |
| `Mapped[bool]` | BOOLEAN | 布尔值 |
| `Mapped[datetime]` | DATETIME | 日期时间 |
| `Mapped[str] = mapped_column(default="")` | VARCHAR | 带默认值的字符串 |

**mapped_column 常用参数：**

| 参数 | 含义 | 示例 |
|---|---|---|
| `primary_key=True` | 主键 | `id: Mapped[int] = mapped_column(primary_key=True)` |
| `default=value` | 默认值 | `priority: Mapped[int] = mapped_column(default=1)` |
| `nullable=True` | 可为空 | `desc: Mapped[str \| None]`（只用类型注解即可） |
| `unique=True` | 值唯一 | `email: Mapped[str] = mapped_column(unique=True)` |
| `index=True` | 创建索引 | `title: Mapped[str] = mapped_column(index=True)` |

### 2.4 Session（会话）

Session 是**单次数据库对话**。每次请求创建一个 Session，结束时关闭。

```python
from sqlalchemy.orm import Session

# 创建会话
session = Session(engine)

# 操作数据库
task = Task(title="买菜")
session.add(task)
session.commit()  # 提交 → 真正写入数据库

# 关闭会话
session.close()
```

**Session 的三个核心方法：**

| 方法 | 作用 | 类比 |
|---|---|---|
| `session.add(obj)` | 将对象加入会话（准备插入） | 把商品放进购物车 |
| `session.commit()` | 提交事务（写入数据库） | 结账付款 |
| `session.rollback()` | 回滚事务（撤销操作） | 清空购物车不买了 |
| `session.close()` | 关闭会话（释放连接） | 离开超市 |

**Session 对象的状态：**

```
New (新建)     → add() →  Pending (待处理)  → commit() →  Flushed (已写入)
Detached (分离) ← close() ← Persistent (持久化) ←
```

| 状态 | 含义 | 在数据库中？ | 被 Session 跟踪？ |
|---|---|---|---|
| **Transient** | 新建的对象，未关联 Session | ❌ | ❌ |
| **Pending** | 已 add()，未 commit() | ❌ | ✅ |
| **Persistent** | 已 commit()，在数据库中 | ✅ | ✅ |
| **Detached** | Session 关闭后 | ✅ | ❌ |

## 三、双层模型设计原则

### 3.1 为什么要分两层

把"数据库模型"和"接口模型"分开，就像**餐厅的后厨和菜单**：

| 层 | 类比 | 职责 | 特点 |
|---|---|---|---|
| **DB 模型** (SQLAlchemy) | 后厨食材 | 对应数据库表结构 | 内部使用，可能含敏感字段 |
| **接口模型** (Pydantic) | 菜单菜品 | 对应 API 请求/响应格式 | 对外暴露，可定制显示 |

### 3.2 两层模型的对比

```python
# ===== DB 模型（SQLAlchemy）— 后厨食材 =====
# 敏感字段、内部字段都在这里
class Task(Base):
    __tablename__ = "tasks"

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str]
    description: Mapped[str | None]
    priority: Mapped[int] = mapped_column(default=1)
    is_deleted: Mapped[bool] = mapped_column(default=False)  # 软删除标记

# ===== 接口模型（Pydantic）— 菜单菜品 =====
# 只暴露给客户端需要的字段
class TaskResponse(BaseModel):
    id: int
    title: str
    description: str | None = None
    priority: int = 1

    model_config = ConfigDict(from_attributes=True)
    # ↑ 这个配置让 Pydantic 能从 ORM 对象自动读取属性
```

### 3.3 from_attributes 配置

`from_attributes=True` 是连接 DB 模型和接口模型的桥梁：

```python
from pydantic import BaseModel, ConfigDict

class TaskResponse(BaseModel):
    id: int
    title: str
    priority: int

    model_config = ConfigDict(from_attributes=True)

# ===== 使用 =====
# DB 模型实例（从数据库查出来的）
db_task: Task = session.query(Task).first()

# 一行代码转成接口模型（自动读取属性）
response = TaskResponse.model_validate(db_task)
# → {"id": 1, "title": "买菜", "priority": 3}
```

**`model_validate()` vs `model_dump()`：**

| 方法 | 作用 | 典型场景 |
|---|---|---|
| `model_validate(obj)` | 从 ORM 对象创建 Pydantic 模型 | 查数据 → 响应给客户端 |
| `model_dump()` | 把 Pydantic 模型转成字典 | 创建 ORM 对象时：`Task(**data.model_dump())` |

### 3.4 完整的分层 CRUD 模式

```python
# ----- DB 模型（SQLAlchemy）-----
class Task(Base):
    __tablename__ = "tasks"
    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str]
    description: Mapped[str | None]
    priority: Mapped[int] = mapped_column(default=1)

# ----- 接口模型（Pydantic）-----
class TaskCreate(BaseModel):
    title: str = Field(..., min_length=1, max_length=50)
    description: str | None = None
    priority: int = Field(default=1, ge=1, le=5)

class TaskResponse(BaseModel):
    id: int
    title: str
    description: str | None = None
    priority: int
    model_config = ConfigDict(from_attributes=True)

# ----- 业务层 -----
def create_task(db: Session, data: TaskCreate) -> TaskResponse:
    # 接口模型 → DB 模型
    db_task = Task(**data.model_dump())
    db.add(db_task)
    db.commit()
    db.refresh(db_task)  # 刷新，获取数据库生成的值（如 id）
    # DB 模型 → 接口模型
    return TaskResponse.model_validate(db_task)
```

### 3.5 `db.refresh()` 的作用

`refresh()` 从数据库中重新读取对象的当前值：

```python
db_task = Task(title="买菜")
db.add(db_task)
db.commit()
# 此时 db_task.id 还是 None，因为 id 是数据库自动生成的
db.refresh(db_task)
# 此时 db_task.id 才是实际的值（比如 1）
```

**交互式文档中的响应模型：**

使用 `response_model` 参数后，`/docs` 页面会自动展示响应体的 JSON 格式：

```python
@router.post("/", response_model=TaskResponse, status_code=201)
def create_task(task_data: TaskCreate, db: Session = Depends(get_db)):
    ...
```

## 四、会话生命周期与依赖注入

### 4.1 问题的起源

每个请求都需要一个数据库会话。如果不管理好，会出两个问题：

| 问题 | 后果 |
|---|---|
| 连接泄露 | 请求结束会话没关，连接池被耗尽 |
| 事务冲突 | 多个请求共享同一个会话，数据错乱 |

### 4.2 标准解决方案：yield 依赖

FastAPI 的 `Depends` 支持 `yield` 关键字，实现"请求开始创建会话，请求结束自动关闭"：

```python
from collections.abc import Generator
from sqlalchemy.orm import Session

def get_db() -> Generator[Session, None, None]:
    """每个请求创建独立的数据库会话，请求结束自动关闭。"""
    db = Session(engine)        # 请求来了 → 创建会话
    try:
        yield db                # 把会话注入给接口使用
    finally:
        db.close()              # 请求结束 → 关闭会话
```

### 4.3 yield 的执行流程

```
客户端请求
  │
  ▼
get_db() 开始执行
  ├── db = Session(engine)       ← 创建会话
  ├── yield db                   ← 把 db 注入给接口
  ▼
接口函数使用 db 进行 CRUD
  ├── db.add(...)
  ├── db.commit(...)
  ▼
接口函数返回响应
  │
  ▼
get_db() 从 yield 后继续执行
  └── finally: db.close()        ← 自动关闭会话
```

**关键理解：**
- `yield` 像是一个"暂停点"——把会话交给接口，等接口用完了再回来执行后面的代码
- `finally` 保证无论接口是否报错，`db.close()` 都会执行
- 这就是 **请求 → 响应 = 创建 → 关闭** 的一一对应关系

### 4.4 在接口中使用

```python
from fastapi import Depends
from sqlalchemy.orm import Session

# 定义依赖（通常放在 dependencies/ 或 database.py 中）
def get_db():
    db = Session(engine)
    try:
        yield db
    finally:
        db.close()

# 接口中注入
@router.get("/tasks")
def read_tasks(db: Session = Depends(get_db)):
    tasks = db.query(Task).all()
    return tasks
```

### 4.5 常见错误

```python
# ❌ 错误：在函数内部创建 engine
def get_db():
    engine = create_engine("sqlite:///./tasks.db")  # 每次请求都创建引擎！
    db = Session(engine)
    ...

# ✅ 正确：engine 是模块级的，只创建一次
engine = create_engine("sqlite:///./tasks.db")

def get_db():
    db = Session(engine)
    ...
```

## 五、事务管理与 CRUD 实战

### 5.1 事务的 ACID 特性

| 特性 | 含义 | 类比 |
|---|---|---|
| **A**tomicity（原子性） | 事务中的所有操作要么全成功，要么全不成功 | 转账：扣钱和加钱必须同时完成 |
| **C**onsistency（一致性） | 事务前后数据状态必须合法 | 余额不能为负数 |
| **I**solation（隔离性） | 并发事务互不干扰 | 两个人同时改同一条数据 |
| **D**urability（持久性） | 事务提交后数据不会丢失 | 钱转出去了就不会凭空消失 |

### 5.2 commit 和 rollback

```python
def create_task(db: Session, data: TaskCreate):
    try:
        db_task = Task(**data.model_dump())
        db.add(db_task)
        db.commit()         # ✅ 提交事务 — 真正写入数据库
        db.refresh(db_task)  # 获取数据库生成的值（id）
        return db_task
    except Exception:
        db.rollback()       # ❌ 出错了，撤销所有未提交的操作
        raise               # 重新抛出，让全局异常处理器处理
```

**commit 和 rollback 的核心规则：**

| 操作 | 需要 commit？ | 说明 |
|---|---|---|
| `db.add()` → `db.commit()` | ✅ 必须 commit | 不 commit 数据不会写入 |
| `db.query().all()` | ❌ 不需要 | 查询不修改数据 |
| `db.delete(obj)` → `db.commit()` | ✅ 必须 commit | 删除也是修改 |
| `db.rollback()` | 出错时调用 | 撤销本次会话所有未提交的操作 |

### 5.3 五种 CRUD 操作

```python
# ----- 增（Create）-----
task = Task(title="买菜", priority=3)
db.add(task)
db.commit()
db.refresh(task)

# ----- 查（Read）- 三种方式 -----
# 查全部
tasks = db.query(Task).all()

# 按条件查
task = db.query(Task).filter(Task.id == 1).first()

# 按主键查（最常用）
task = db.get(Task, 1)

# ----- 改（Update）-----
task = db.get(Task, 1)        # 先查出来
task.title = "买水果"           # 修改属性
db.commit()                   # 提交（不需要再 add，因为对象已被 Session 跟踪）
db.refresh(task)              # 刷新

# ----- 删（Delete）-----
task = db.get(Task, 1)        # 先查出来
db.delete(task)                # 删除
db.commit()                   # 提交
```

### 5.4 filter 条件速查

```python
# 等值查询
db.query(Task).filter(Task.title == "买菜")

# 模糊查询（LIKE）
db.query(Task).filter(Task.title.contains("菜"))

# 范围查询
db.query(Task).filter(Task.priority >= 3)

# 多条件 AND
db.query(Task).filter(Task.title == "买菜", Task.priority >= 3)

# 多条件 OR
from sqlalchemy import or_
db.query(Task).filter(or_(Task.title == "买菜", Task.priority >= 3))

# 排序
db.query(Task).order_by(Task.priority.desc())  # 降序
db.query(Task).order_by(Task.id.asc())          # 升序

# 分页
db.query(Task).offset(0).limit(10).all()
```

### 5.5 FastAPI + SQLAlchemy CRUD 完整代码

```python
from fastapi import Depends
from sqlalchemy.orm import Session
from app.database import get_db
from app.schemas.task import TaskCreate, TaskResponse
from app.models.task import Task

@router.post("/", response_model=TaskResponse, status_code=201)
def create_task(data: TaskCreate, db: Session = Depends(get_db)):
    db_task = Task(**data.model_dump())
    db.add(db_task)
    db.commit()
    db.refresh(db_task)
    return db_task

@router.get("/", response_model=list[TaskResponse])
def read_tasks(db: Session = Depends(get_db)):
    return db.query(Task).all()

@router.get("/{task_id}", response_model=TaskResponse)
def read_task(task_id: int, db: Session = Depends(get_db)):
    task = db.get(Task, task_id)
    if not task:
        raise HTTPException(status_code=404, detail="任务未找到")
    return task

@router.put("/{task_id}", response_model=TaskResponse)
def update_task(task_id: int, data: TaskCreate, db: Session = Depends(get_db)):
    task = db.get(Task, task_id)
    if not task:
        raise HTTPException(status_code=404, detail="任务未找到")
    for key, value in data.model_dump().items():
        setattr(task, key, value)
    db.commit()
    db.refresh(task)
    return task

@router.delete("/{task_id}", status_code=204)
def delete_task(task_id: int, db: Session = Depends(get_db)):
    task = db.get(Task, task_id)
    if not task:
        raise HTTPException(status_code=404, detail="任务未找到")
    db.delete(task)
    db.commit()
    return
```

**关于 update 的写法说明：**

```python
# 这种"循环赋值"写法，比手动一个一个赋值更通用：
for key, value in data.model_dump().items():
    setattr(task, key, value)

# 等效于手动写（但新增字段时不用改这里的代码）：
task.title = data.title
task.description = data.description
task.priority = data.priority
```

## 六、附录：数据库配置速查

### 6.1 常见数据库连接地址

| 数据库 | 连接地址 | 安装依赖 |
|---|---|---|
| SQLite | `sqlite:///./data.db` | 内置（无需安装） |
| SQLite（内存） | `sqlite:///:memory:` | 内置 |
| MySQL | `mysql+pymysql://user:pass@localhost:3306/dbname` | `pip install pymysql` |
| PostgreSQL | `postgresql://user:pass@localhost:5432/dbname` | `pip install psycopg2-binary` |

### 6.2 数据库配置模块示例

```python
# app/database.py
from sqlalchemy import create_engine
from sqlalchemy.orm import DeclarativeBase, Session

# 1. 创建引擎（模块级别，只创建一次）
DATABASE_URL = "sqlite:///./tasks.db"
engine = create_engine(DATABASE_URL, echo=True)

# 2. 创建 Base（所有模型的父类）
class Base(DeclarativeBase):
    pass

# 3. 依赖函数（每个请求一个会话）
def get_db():
    db = Session(engine)
    try:
        yield db
    finally:
        db.close()

# 4. 建表函数（项目启动时调用）
def init_db():
    Base.metadata.create_all(engine)
```

### 6.3 项目集成清单

将数据库整合到 FastAPI 项目中需要完成以下步骤：

```
1. app/database.py           ← 创建 Engine、Base、get_db 依赖
2. app/models/task.py        ← 定义 SQLAlchemy DB 模型
3. app/schemas/task.py       ← 定义 Pydantic 接口模型（加 ConfigDict）
4. app/routers/task.py       ← 注入 get_db，DB 模型 ↔ 接口模型转换
5. app/main.py               ← 启动时调用 init_db() 建表
```
